# AI Assistant Rules for IntelliJ IDEA

## 项目简介

本项目包含用于 **IntelliJ IDEA 2026.1.1** 中 AI Assistant 的规则文件。这些规则基于"输出措辞工程"（Output Phrasing Engineering）方法论，旨在系统性地提升 AI 编码助手在 Agent 模式下的输出质量。

## 核心理念

自回归语言模型的每一个 token 都在影响后续 token 的概率分布。通过四个层面的协同作用，可以在不修改模型权重的前提下提升输出质量：

| 层级 | 作用 | 描述 |
|------|------|------|
| 第零层 | 人设与行文风格 | 全局性的推理偏置，持续影响所有 token 的概率分布 |
| 第一层 | 强制可见思维链 + 措辞替换 | 在推理起点锚定高质量方向 |
| 第二层 | 元认知决策协议 | 遇到意外问题时的完整决策流程，防止静默降级 |
| 第三层 | 硬规则兜底 | 绝对禁止的降级行为，最后一道防线 |

## 文件说明

| 文件 | 描述 |
|------|------|
| `output-phrasing-engineering.md` | 输出措辞工程的完整理论框架，包含全局架构、思维链格式、措辞替换映射表、教学示例等 |
| `rules-agent-mode.md` | Agent 模式的具体规则实现，包含人设定义、三阶段流程（侦察→分析→执行）、措辞替换规则和完整示例 |

## 核心流程

所有涉及代码修改的任务必须遵循三阶段流程，**不存在"简单任务可以跳过"的例外**：

```
用户发出任务
      │
      ▼
┌─────────────────────────────────┐
│  阶段一：Reconnaissance（侦察）  │
│  读取相关文件、搜索代码库        │
│  → 先搞清楚"现在是什么样"        │
├─────────────────────────────────┤
│  阶段二：Analysis（分析）         │
│  基于侦察结果做方案设计          │
│  → 知道了现状，才能做合理决策     │
├─────────────────────────────────┤
│  阶段三：Execution（执行）        │
│  按 execution_plan 调用工具      │
│  → 按图施工                      │
└─────────────────────────────────┘
```

## 措辞替换机制

当 AI 即将产出降级短语时，替换为高质量表达：

| 降级类型 | 示例（即将说） | 替换为 |
|----------|----------------|--------|
| 捷径导向型 | "最简单的做法是" | "综合评估后，推荐的方案是" |
| 放弃方案型 | "代码太长了，我换个方式" | "实现内容较多，我分为N个部分输出，第1部分是" |
| 虚假完成型 | "// ... 其他代码保持不变" | [直接输出完整的修改上下文] |
| 过度抽象型 | "你可以根据需要扩展" | "完整实现已包含扩展点，具体是" |
| 自我降权型 | "我这里做一个简化版" | "完整实现如下" |
| 逃避复杂度型 | "这个问题比较复杂，建议" | "这个问题可以拆解为以下子问题" |
| Agent 模式专属 | "其他文件应该不需要改" | "我先搜索确认是否有其他文件需要同步修改" |
| 思维链内部省略型 | "degradation_check: 全部通过" | 逐条写出每个检查项的 YES/NO 和判断理由 |

## 快速开始

1. 将规则文件放置到 IntelliJ IDEA 的 AI Assistant 规则目录
2. 确保规则文件头部包含 `apply: always` 元数据（已设置）
3. 重启或刷新 AI Assistant，规则将自动生效

## 主要特性

- 防止 AI "偷懒"的措辞替换机制
- 强制可见思维链（XML 标签格式），防止跳过思考
- 元认知决策协议（`<decision_point>`），处理执行阶段的意外情况
- 完整的对话式教学示例（Few-Shot Learning）
- 支持 Agent 模式（工具调用）和 Chat 模式（直接输出代码）

## 适用场景

| 模式 | 工具示例 | 行为特征 |
|------|----------|----------|
| Agent 模式 | CC / CodeBuddy / Cursor | AI 通过工具调用修改代码，不直接输出完整代码 |
| Chat 模式 | ChatGPT / API 对话 | AI 直接在回复中输出完整代码，用户复制粘贴 |

两种模式的思维链和降级检测设计相同，但实现阶段的行为规则不同。
